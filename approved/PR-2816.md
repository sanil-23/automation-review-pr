## AI Summary

**What it does**: Fixes a Windows-specific startup failure (Sentry OPENHUMAN-TAURI-500, 3 events, v0.56.0) where port 7788 landing in an OS-excluded range (Hyper-V/WinNAT/WSL2/Docker reserved ranges) returned WSAEACCES (os error 10013) on bind. That error fell into the catch-all `BindFailed` arm, preventing the embedded core from starting entirely. The fix introduces `is_port_excluded_bind_error`, routes WSAEACCES to the existing fallback port pool (7789–7798) while skipping the stale-listener takeover probe (nothing is listening on an excluded port), and extracts `pick_fallback_port` as a reusable function.

**Breaking risk**: Zero. Touches only port-selection internals. All non-WSAEACCES paths are byte-for-byte unchanged. Existing AddrInUse tests remain green.

**Security risk**: Zero. No auth, no data, no injection surface. Connectivity-layer internals only.

**Bottom line**: Safe to merge — targeted Windows bug fix, well-tested, CI fully green.

## AI Summary
- **What it does**: Adds `ExpectedErrorKind::FilesystemUserPathInvalid` to the observability classifier so RPC-level filesystem path validation errors (e.g. `vault_create` called with a non-existent `root_path`) are demoted from Sentry events to `tracing::warn!` breadcrumbs. Fixes Sentry issue TAURI-RUST-4QH and also preempts the symmetric `http_host::path_utils` shape. The `warn!` arm logs only `domain`/`operation`/`kind` — no raw message, no user filesystem path — mirroring the `LoopbackUnavailable` arm's "metadata over raw text" principle.
- **Breaking risk**: Zero — purely additive: new enum variant, new predicate, new dispatcher arm, new tests. No changed signatures, no modified behavior outside the error classification path.
- **Security risk**: Zero — net positive: prevents user filesystem paths (which embed OS usernames) from reaching Sentry via breadcrumbs.
- **Bottom line**: Safe to merge.

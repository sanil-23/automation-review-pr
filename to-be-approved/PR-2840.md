- **Breaking risk**: Zero — only affects malformed histories that already caused 400 errors. Well-formed histories pass through unchanged. Trait signatures, RPC schemas, and dispatcher output for paired histories are untouched.
- **Security risk**: Zero — no new network surface, no new inputs trusted, no auth path touched.
- **Bottom line**: Safe to merge pending test coverage for the turn.rs path and CodeRabbit's tool_call_id validation concern.

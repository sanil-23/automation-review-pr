## AI Summary
- **What it does**: Fixes HTTP 400 errors on turn 2+ when using thinking models (DeepSeek-R1, Qwen3, GLM-4) via the OpenAI-compatible provider. The fix propagates `reasoning_content` from the API response through `ChatResponse`, stores it in `ChatMessage.extra_metadata`, and echoes it back in the next request via `convert_messages_for_native`. Standard providers are unaffected because `skip_serializing_if = "Option::is_none"` omits the field when absent.
- **Breaking risk**: Low. `ChatResponse` gains a new public field but also derives `Default`, so external code can use struct update syntax. Internal usages are all updated in this PR.
- **Security risk**: Zero. The field is a plain string from the provider API, stored and echoed verbatim — no execution or injection surface.
- **Bottom line**: Safe to merge. CI failure is on "Build & smoke-test core image" (Docker), unrelated to these Rust-only changes.

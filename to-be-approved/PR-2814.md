## AI Summary
- **What it does**: Adds `is_backend_auth_failure()` and `publish_backend_session_expired()` helpers in `ops.rs`, then wires the former as the first guard in `chat_with_system`'s HTTP error chain. Backend 401/403 from the OpenHuman provider now publishes `DomainEvent::SessionExpired` (driving reauth + halting downstream LLM work via scheduler-gate) and skips Sentry instead of falling through to `report_error`. Direct fix for TAURI-RUST-N (332 Sentry events). Third-party BYO-key 401/403 are unaffected.
- **Breaking risk**: Zero. Internal error classification change; no exported functions, public APIs, shared types, DB schemas, or config formats changed.
- **Security risk**: Zero. `sanitize_api_error` applied to the reason string before it reaches `SessionExpired.reason`; no new data surfaces, no secrets introduced.
- **Bottom line**: Safe to merge.

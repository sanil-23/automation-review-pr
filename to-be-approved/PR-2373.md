## Quick Summary
*Generated: 2026-05-22T08:06:08.050Z*

**What it does** — When a user asks the agent to "send an email" (or use any connected integration), the app checks whether that integration is actually usable. Previously, every non-active connection got the same generic "not authorized yet" message — even if Gmail was mid-OAuth, had an expired token, or had a failed handshake. This confused users because Settings showed Gmail as connected while the agent said it wasn't. This PR makes the gate inspect the real upstream connection status (INITIATED, EXPIRED, FAILED, etc.) and returns a specific, actionable message for each case — "finish your OAuth flow," "your token expired, reconnect," etc.

**Safety & Breaking concerns** — **Low**

- **No security issues.** No new IO, no new auth surfaces, no user input rendered unsanitized. The `non_active_status` string comes from Composio's backend, not user input, and is only inserted into agent-facing prompt text (not HTML/JS).
- **No breaking changes.** `ConnectedIntegration` gained one `Option<String>` field — all 14 in-repo construction sites are updated. The happy path (ACTIVE connections) is completely untouched. The `None` / truly-disconnected branch preserves the exact legacy wording.
- **Edge cases handled well.** Case-insensitive matching, whitespace trimming (blank collapses to `None` branch), unknown statuses quoted verbatim rather than swallowed, priority ordering when multiple non-active rows exist for the same slug, and ACTIVE slugs are excluded from the non-active map.
- **Test coverage is solid.** 7 new focused tests covering all branches including case insensitivity, whitespace, unknown statuses, and the legacy fallback. Existing tests updated with `non_active_status: None`.
- **Minor note:** the `describe_unconnected_state` function is `pub(crate)` — appropriately scoped. The status priority logic in `ops.rs` is clean and local (no global state changes).

**Bottom line** — Safe to merge: it's a targeted, well-tested fix that only changes agent-facing error copy for non-active integrations, with zero impact on the happy path.

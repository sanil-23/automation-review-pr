## AI Summary
**What it does**: Adds a per-model health comparison panel to Settings > Developer Options. Operators can see quality scores, hallucination rates, cost per million tokens, and a recommended action badge for each model in their registry. Supports sorting, filtering by status, and a one-click model swap modal (Apply is UI-only, backend swap is a documented follow-up).
**Breaking risk**: Zero — all new config fields use #[serde(default)], no existing APIs modified, controller registry additions are additive.
**Security risk**: Zero — RPC call goes through `callCoreRpc` relay; no raw network access from renderer; config read-only; no secrets or auth surfaces.
**Bottom line**: Safe to merge once CI is green — all code issues resolved, architecture is clean, tests are solid.

## AI Summary

**What it does**: Adds Querit as a new web search engine provider alongside Managed, Parallel, Brave, Seltz, and SearXNG. Users configure a Querit API key in Settings > Search, and when Querit is selected, the core calls the Querit POST API for web search with support for site, time range, country, and language filters. Includes full i18n support across 10 languages and Tauri settings UI integration.

**Breaking risk**: Zero. New optional search provider with no changes to existing search engine paths or public APIs.

**Security risk**: Medium-High. Two issues:
1. Querit API key is not included in secret encryption/decryption paths — will persist in plaintext when secrets.encrypt=true.
2. Error responses from Querit API are included in error messages; if the response echoes user queries or request context, it will leak into logs.

**Bottom line**: Not safe to merge — Security issues must be fixed (missing secret encryption, error data exposure). Once addressed, the code quality and test coverage are solid.

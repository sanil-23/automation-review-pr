## AI Summary

**What it does**: Adds an `Arc<tokio::sync::OnceCell<PathBuf>>` field to `SecurityPolicy` that lazily caches the canonical form of `workspace_dir`. Previously `validate_path` and `validate_parent_path` each called `tokio::fs::canonicalize(&workspace_dir)` inline on every invocation — one syscall per path validation across a potentially hot code path. After the first call the result is memoized; a fresh policy (`from_config`) gets a fresh empty cell, so config updates naturally invalidate.

**Breaking risk**: Low. Change is additive — one new field on an existing struct. All within-crate struct literal callers use `..Default::default()` spread and are unaffected. The visibility bug (see below) does break external crates.

**Security risk**: Zero. Pure performance optimization; fallback to raw `workspace_dir` on canonicalize failure preserves existing semantics exactly.

**Bottom line**: Not safe to merge as-is — CI failing due to `pub(crate)` visibility bug that breaks `mouse_smoke` example compilation.

## AI Summary
**What it does**: Implements 6 new capability domains (voice assistant, guided flows, live captions, voice actions, operator inbox, chat-with-data) as Rust modules following the controller-registry pattern. Each module includes engine logic, RPC handlers, types, schemas, and unit tests. All wired into the core controller registry and capability catalog.

**Breaking risk**: Low. PR is stated as additive only — no existing modules modified except core/all.rs (adding registrations) and about_app/catalog.rs (adding entries). No exported function signatures changed.

**Security risk**: Medium. Multiple issues flagged by CodeRabbit: path traversal via transcript IDs in persist.rs, SSRF risk in webhooks (mitigated by IP validation but DNS rebinding not blocked), PII logged in operator_inbox (email subjects/addresses), raw SQL logged in db_connector.rs. The webhook SSRF validation does block private IPv4/IPv6 ranges and loopback but is vulnerable to DNS rebinding at request time.

**Bottom line**: Not safe to merge — CI failing, Coverage Gate failing, and multiple CodeRabbit major findings (path traversal, PII in logs, processing lock safety, RpcOutcome pattern deviation) need resolution first.

## AI Summary
- **What it does**: Fixes multiple runtime bugs in the Meet bot pipeline — silent caption listener (heartbeat + bridge probe), broken CC auto-enable from Meet UI changes (keyboard shortcut fallback, broadened selectors), echo loop from incoming WebRTC audio (HTMLMediaElement + RTCPeerConnection + Web Audio muting), mascot's WebGL canvas not rendering when fully off-screen (moved to left:0 opacity:0), and mascot mouth not animating during TTS playback (new SpeakingTracker in speak_pump + Tauri events to frontend). Also adds owner-gated privacy lock (only call owner triggers tool calls), barge-in support (flush_pending), and a persistent AGENT_CACHE for multi-turn session memory.
- **Breaking risk**: Low. `owner_display_name` has `#[serde(default)]` on Rust side; `ownerDisplayName` is optional in TS type.
- **Security risk**: Zero — owner-gated wake word is a positive improvement.
- **Bottom line**: Safe to merge once CI passes and RTCPeerConnection addEventListener wrapper is addressed.

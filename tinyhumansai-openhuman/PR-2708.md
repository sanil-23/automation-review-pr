## AI Summary

**What it does**: Stamps a `QuoteOwner { thread_id, client_id }` — derived from the `APPROVAL_CHAT_CONTEXT` task-local — onto every `PreparedTransaction` at prepare time, then gates `execute_prepared` to only allow execution by the caller whose owner matches the prepare-time stamp. Closes a vulnerability where a co-channel observer in a shared multi-member channel could read a broadcast `quote_id` and execute another user's prepared wallet transaction from their own (now per-sender-isolated, post-#2331) agent session.

**Breaking risk**: Low. `owner` is annotated `#[serde(skip_serializing_if = "Option::is_none")]` so existing wire format is unchanged for no-context callers. Field is `pub(crate)` and not part of any external API. The equality check is `None == None` so CLI/direct-RPC flows are unaffected.

**Security risk**: Low (with one fix pending). The PR addresses a Medium vulnerability. Remaining concern: the `#[serde(skip_serializing_if = "Option::is_none")]` annotation on `PreparedTransaction.owner` still serializes `thread_id`/`client_id` as `threadId`/`clientId` when the chat context is set — internal session identifiers leak over the wire. Should be `#[serde(skip_serializing)]`.

**Bottom line**: Not safe to merge yet — CodeRabbit's serialization fix is required, CI is failing on multiple checks.

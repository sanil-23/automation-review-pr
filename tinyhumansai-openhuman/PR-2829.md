## AI Summary

**What it does**: Adds a `patch_apprun_sharun_cwd` function to the release script that injects `cd "$APPDIR" &&` into the AppRun shell script before the final `exec "$@"` line. This ensures the working directory is always the AppDir when sharun runs, fixing a bug where AppImages fail to launch from any directory other than the AppDir itself because sharun resolves preload/library paths relative to CWD, not AppDir.

**Breaking risk**: Zero. This is a release-pipeline bash script change only; no API, config, or user-facing changes.

**Security risk**: Low. The function properly validates input (ELF check, idempotency guard), uses temp files to avoid truncation, and is guarded by the existing `uses_sharun_launcher` check. However, the grep patterns flagged by CodeRabbit are loose enough to cause idempotency/validation false positives.

**Bottom line**: Safe conceptually, but correctness issues in the grep patterns need fixing. Once tightened, this is safe to merge.

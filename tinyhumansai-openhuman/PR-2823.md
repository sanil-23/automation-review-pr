## AI Summary
**What it does**: Adds a per-model health comparison panel to Settings > Developer Options. Operators can see quality scores, hallucination rates, cost per million tokens, and a recommended action badge for each model in their registry. Supports sorting, filtering by status, and a one-click model swap modal.
**Breaking risk**: Zero — all new config fields use #[serde(default)], no existing APIs modified, new route is additive.
**Security risk**: Low — new endpoint is behind Bearer token auth. Auth logic is inline copy-paste rather than shared middleware (inconsistent pattern) but functionally correct.
**Bottom line**: Not safe to merge — "Apply Replacement" is a no-op, there's no candidate selection mechanism in the modal, the backend returns only placeholder metrics making the dashboard misleading, and the raw fetch() violates the established RPC relay pattern. CI also still pending.

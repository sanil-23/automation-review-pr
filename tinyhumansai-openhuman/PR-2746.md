## AI Summary

**What it does**: Adds a new `claude_agent_sdk` provider type that routes inference requests through the Claude CLI subprocess (`claude -p`) instead of the HTTP API, allowing users to consume Agent SDK credit (Pro/Max/Team plans) instead of API quotas. Includes configuration table, factory dispatch, doctor health check, and comprehensive unit tests. Feature is disabled by default and requires no config migration.

**Breaking risk**: Zero — purely additive. No exported functions changed, no shared types modified, config disabled by default.

**Security risk**: Low — proper subprocess lifecycle management. Concurrent stderr draining prevents pipe buffer stalls, 120s/30s timeouts prevent indefinite hangs, NDJSON payloads redacted in logs (only line_len logged, never raw content), stderr captured for error context, no shell injection vectors.

**Bottom line**: Safe to merge once CI fully passes.

## PR Description
The PR fixes a bug where the "Your included budget is complete" banner and `isAtLimit` / `isRateLimited` signals fired even when the user had routed all three chat workloads (reasoning, agentic, coding) to their own provider (OpenRouter, OpenAI, Anthropic key, or local Ollama). The hook `useUsageState` previously only consulted `cycleBudgetUsd` / `remainingUsd` from the `/teams` endpoint and never checked the user's actual LLM routing.

**Fix approach:**
1. Add `loadAISettings()` to `fetchUsageData`'s `Promise.all` — independently caught, failure → `null` (conservative).
2. Derive `allChatWorkloadsRoutedAway` — true iff every `CHAT_WORKLOADS` entry (`reasoning`, `agentic`, `coding`) has `ref.kind !== 'openhuman'`.
3. Gate `isBudgetExhausted`, `shouldShowBudgetCompletedMessage`, `isRateLimited` on `!allChatWorkloadsRoutedAway`.
4. Export `allChatWorkloadsRoutedAway` on `UsageState` for other surfaces.

**Tests**: 9 passed (6 legacy + 3 new) — happy path, partial routing, fetch failure conservative fallback.

## Impact Scan

- **Added**: `allChatWorkloadsRoutedAway: boolean` — new field on the `UsageState` interface
- **Changed semantics**: `isBudgetExhausted`, `shouldShowBudgetCompletedMessage`, `isRateLimited` can now be `false` even when budget is exhausted (if routed away). `isAtLimit` follows since it's derived from the other two.

### Consumer impact
- **Home.tsx**: Uses `isRateLimited` and `shouldShowBudgetCompletedMessage` — both now correctly suppressed when routed away. No code changes needed.
- **Conversations.tsx**: Uses `isAtLimit` (blocks send), `isBudgetExhausted` (modal), `shouldShowBudgetCompletedMessage` (alert), `isRateLimited` (alert) — all now correctly suppressed when routed away. No code changes needed. The send-blocking via `isAtLimit` is the most critical consumer — and it correctly benefits from the suppression.

### Cache shape change
The `_cache` type changed from `{ teamUsage, currentPlan }` to `{ teamUsage, currentPlan, aiSettings }`. This is purely internal — no external consumers of the cache. The cache is only populated when `teamUsage && currentPlan` are both non-null, so `aiSettings` can be null in a cached entry, which is fine.

N/A — no signature changes that break callers. The new field is additive.

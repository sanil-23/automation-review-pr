## PR Description

Four `tree_jobs_worker` tasks race into `memory::tree::store::with_connection` on cold start, triggering ~19k Sentry events/day across three SQLite WAL/SHM bootstrap codes:

| Sentry ID | Events | Error |
|---|---|---|
| OPENHUMAN-TAURI-HH | 14168 | `Failed to initialize memory_tree schema: disk I/O error: Error code 1546` |
| OPENHUMAN-TAURI-ZM | 3013 | `Failed to initialize memory_tree schema: disk I/O error: Error code 4874` |
| OPENHUMAN-TAURI-MB | 2177 | `Failed to initialize memory_tree schema: unable to open database file: Error code 14` |

Solution: Gate `apply_schema` behind a process-wide `Mutex<HashSet<PathBuf>>` so only the first caller per DB path runs WAL setup + SCHEMA + ALTER TABLE migrations; subsequent callers skip. Retry open+init up to 3x on the three transient cold-start codes (14/1546/4874) with 50→150→500ms backoff. Regression test spawning 8 threads against a cold tempdir DB.

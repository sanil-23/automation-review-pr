## AI Summary

**What it does**: Inserts an `is_native_tool_schema_unsupported` guard in `stream_native_chat`'s non-2xx handler, between the provider-config rejection check and the Sentry gate. When an Ollama model rejects tool definitions with a 400, the function logs at `info!` and returns the error without triggering Sentry — the existing retry loop in `chat()` re-issues the request without tools. Two unit tests cover the detection logic and the retry error-string matching.

**Breaking risk**: Zero — change is confined to an `else if` branch in the non-2xx error path; only activates on specific 400 bodies matching known tool-schema rejection phrases. No exports or shared types changed.

**Security risk**: Zero — no auth changes, no secrets, no injection surfaces, no logging of sensitive data.

**Bottom line**: Safe to merge.

## AI Summary
**What it does**: Encodes `reasoning_content` alongside `content` as a JSON blob in the assistant message text field when a thinking model responds, then detects and unpacks that blob on the next turn to reconstruct `NativeMessage` with `reasoning_content` populated for API replay. Fixes 400 errors from DeepSeek-R1/Qwen3 on multi-turn conversations.
**Breaking risk**: Medium — JSON-blob encoding changes the data shape of `ProviderChatResponse::text` for thinking-model turns; downstream consumers (UI, summarizer, export) receive raw JSON instead of plain message text.
**Security risk**: Zero — no auth, secrets, or injection surfaces touched.
**Bottom line**: Not safe to merge — JSON-blob in text field leaks to unrelated consumers, and the `if tool_calls.is_empty()` gate silently breaks agentic flows.

**Findings**:
- [major] compatible.rs:847 — `if tool_calls.is_empty()` gate silently drops reasoning_content for tool-call turns; thinking models routinely emit reasoning alongside tool calls, so agentic flows still 400 after this fix
- [major] compatible.rs:857 — encoding reasoning_content into the `text` field is a cross-cutting data-shape change; UI render path, summarizer, export, subagent transcript extraction will all receive raw JSON instead of plain text for thinking-model turns
- [minor] compatible.rs:596 — naive JSON heuristic (`value.get("reasoning_content")`) misclassifies any stored message whose content happened to be JSON with that key; sentinel prefix needed for unambiguous detection

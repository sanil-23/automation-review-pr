## What it does

This PR teaches the desktop mascot to pick a contextual facial expression after each chat turn completes, instead of always defaulting to "happy." It reads two signals from the `chat_done` event — an explicit `reaction_emoji` (e.g. ✅, 🤔, ⚠️) and fallback keyword matching on the response text — to select one of three post-turn faces: **happy**, **confused**, or **concerned**. When no strong cue exists, it falls back to the previous calm/happy behavior. The same logic applies whether TTS is on or off, and audio decode/playback failures still force "concerned." This is a partial implementation of issue #1144.

**Files changed:** `useHumanMascot.ts` (new `pickConversationAckFace` function + wiring), `useHumanMascot.test.ts` (new test coverage), and the mascot GitBook docs.

## Safety & Breaking concerns

**Breaking risk: Zero**

- **No security surface** — purely UI-side expression logic; no network calls, no user data handling, no auth changes.
- **Backwards compatible** — when no emoji or text cue matches, `pickConversationAckFace` returns `null` and the code falls back to `'happy'`, which is the old behavior. Existing states (thinking, speaking, listening, error) are untouched.
- **Regex is bounded** — the three regexes use simple `\b` word-boundary matches with no backtracking risk or ReDoS exposure.
- **Error path preserved** — TTS synth failures and playback decode errors still land on `'concerned'` via the existing `degraded` flag, and the new outer `catch` block correctly sets `degraded = true` before rethrowing.
- **Test coverage is solid** — 34 tests covering emoji priority, text-cue fallback, null/missing input, TTS success path, and audio failure path. The new outer catch for playback errors (line ~353) has a dedicated test.
- **No new dependencies**, no lockfile changes, no API contract changes.

One minor note: the emoji sets use literal Unicode (e.g. `'⚠️'` vs `'⚠'`), and both variants are included in `CONCERNED_REACTION_EMOJIS`, which correctly handles platform variation.

## Bottom line

Safe to merge — it's a self-contained, well-tested UI enhancement with clean fallback to existing behavior and no external surface area.

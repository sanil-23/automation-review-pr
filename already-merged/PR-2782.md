## AI Summary
- **What it does**: Adds `"operation timed out"` to the `is_network_unreachable_message` classifier in `src/core/observability.rs`. Previously, ETIMEDOUT errors from supervised channel listeners (discord/slack/telegram) fell through every classifier arm into `report_error`, generating ~128 Sentry events that were pure noise — the supervisor already handles these via exponential backoff. Now they route to `ExpectedErrorKind::NetworkUnreachable` and demote to a structured `warn!` log.
- **Breaking risk**: Zero. Pure internal classifier refinement; no exported types, public APIs, DB schemas, or config formats touched.
- **Security risk**: Zero. No auth, no user input, no network surface, no secrets.
- **Bottom line**: Safe to merge.

## AI Summary

**What it does**: Adds image attachment support to desktop chat. Users can click a paperclip button in the composer, select image files (PNG, JPEG, WebP, GIF, BMP), see thumbnail previews with per-item remove buttons, and send images alongside text. Images are encoded as `[IMAGE:<data-uri>]` markers in the message, which the existing Rust backend (`agent/multimodal.rs`) already parses and routes to the inference provider — no backend changes needed. Supports up to 4 images per message, 8 MB each. Validation errors (unsupported type, oversized, count exceeded) surface in the composer error banner. Supports send-with-attachments-only (empty text + images). Attachments clear after send.

**Breaking risk**: Zero. No new exports that could break callers, no changes to existing APIs, no RPC changes, no Redux state reshaping.

**Security risk**: Low. FileReader.readAsDataURL() only processes client-selected files (no network source). Data URIs embedded in message text — no script injection risk. MIME type validation prevents non-image files. Size limit (8 MB) prevents memory exhaustion. No hardcoded secrets, credentials, or PII in logs.

**Bottom line**: Safe to merge — all prior feedback addressed, CI green, solid test coverage, clean architecture, no breaking changes.

## What it does

This PR fixes Docker builds failing with SIGKILL (out-of-memory) on small VPS and CI machines. It switches the default Cargo build profile from `release` (high optimization, high memory) to `ci` (lighter optimization, lower peak memory) and caps Cargo parallelism to 1 job. The binary is copied through a stable `/tmp` path so the profile name doesn't break the final `COPY` step. Operators who want full optimization can still pass `--build-arg CARGO_PROFILE=release`.

Closes issue #2419, where a user reported `docker build` being killed by the OOM killer during compilation on their VPS.

## Safety & Breaking concerns

**Breaking risk: Low**

- **No runtime behavior change** — the binary is the same code, just compiled with slightly less optimization by default. Entrypoint, ports, env vars, and health check are untouched.
- **No security concerns** — no auth, input handling, or network code is modified. This is purely build infrastructure.
- **Profile path assumption**: The `cp "target/${CARGO_PROFILE}/openhuman-core" /tmp/openhuman-core` line assumes Cargo outputs to `target/<profile-name>/`. This is true for named profiles like `ci` and `release`, but would break if someone passed a profile that Cargo maps to a different directory name (e.g., `dev` maps to `target/debug/`). In practice this is fine — `ci` and `release` are the only realistic values, and it's the caller's responsibility to pass a valid profile.
- **Escape hatch preserved** — `--build-arg CARGO_PROFILE=release` restores the old behavior exactly.
- **Dep cache layer could be stale across profile switches** — if someone rebuilds with a different `CARGO_PROFILE` without clearing the Docker cache, the dependency cache layer (built with the old profile) gets reused. Docker's layer caching should handle this correctly since the `ARG` change invalidates downstream layers, but it's worth noting.

## Bottom line

Safe to merge — it's a narrowly scoped Dockerfile change that fixes a real OOM build failure, preserves an opt-in path to full optimization, and touches zero runtime code.

## Quick Summary
*Generated: 2026-05-22T08:42:58.878Z*

**What it does** — Adds an HTTP/SSE transport option to the existing MCP server so external clients (IDEs, other agents) can connect to OpenHuman over the network, not just via stdin/stdout pipes. Running `openhuman-core mcp --transport http` starts a local web server on port 9300 that speaks the standard MCP protocol. Optional bearer token auth protects the endpoint. Stdio mode (the default) is unchanged. This is Phase 1 of issue #1845 — no config file wiring or server-push notifications yet.

**Safety & Breaking concerns** — **Medium** risk.

1. **Infinite loop bug in CLI arg parser.** The `while index < args.len()` loop in `stdio.rs` never increments `index` for simple flags (`-v`/`--verbose`). Running `openhuman-core mcp --verbose` will spin forever at 100% CPU. The `--transport`, `--host`, `--port`, `--auth-token` arms do `index += 2` but there's no fallthrough `index += 1` for single-token flags. This is a ship-blocking bug.

2. **Session memory leak.** HTTP sessions are stored in an unbounded `HashMap` with no TTL or eviction. Clients that don't send `DELETE` leave sessions behind forever. Under sustained use or abuse, this leaks memory without bound.

3. **No rate limiting or connection caps.** The HTTP server accepts unlimited concurrent connections and session creations. A single client can exhaust server resources.

4. **Stateless protocol dispatch loses session tracking.** Every HTTP request calls `protocol::handle_json_value()` which creates a throwaway `McpSession::default()` — so the client-name-based `source_type` tracking (used for observability logs) that stdio gets is lost in HTTP mode. The HTTP layer tracks session IDs for auth routing but never passes session context into the protocol layer.

5. **No breaking changes to existing behavior.** Stdio mode is untouched. The HTTP transport is additive and opt-in. Capability catalog update is cosmetic.

6. **Auth is sound for its scope.** Bearer token check uses constant-time-safe string comparison (Rust `==` on `&str`), rejects early on all endpoints, and the token is never logged.

**Bottom line** — Not safe to merge as-is: the CLI arg parser infinite loop is a blocking defect, and the session leak + missing protocol-level session context are significant gaps that should be addressed before this ships.

## What it does

This PR adds stable `data-testid` attributes to UI components that E2E tests interact with — settings navigation rows, skill cards/tiles, conversation thread rows, chat composer buttons, and debug panels (Webhooks, Memory). It replaces brittle Tailwind class-string selectors with predictable hooks like `settings-nav-account`, `skill-row-calendar`, `thread-row-<id>`, `send-message-button`, etc. It also renames the existing `uninstall-skill-*` testid to `skill-uninstall-*` to match the documented taxonomy from issue #1861.

Four new/updated component test files verify the hooks render correctly.

## Safety & Breaking concerns

**Breaking risk: Zero**

- **No runtime behavior changes** — only `data-testid` attributes added to DOM elements. These are invisible to users and have no effect on styling, event handling, or state.
- **One testid rename** (`uninstall-skill-*` → `skill-uninstall-*`) — the existing test that referenced the old name is updated in the same PR. If any out-of-tree E2E spec used the old `uninstall-skill-*` selector it would break, but the PR description confirms this is the canonical rename and the taxonomy doc governs naming.
- **No new dependencies, no API changes, no schema changes.**
- **No security surface** — `data-testid` attributes don't accept user input and expose no sensitive data.
- **The wrapper `<div>` added around `ChannelTile` and `ComposioConnectorTile`** introduces an extra DOM node in the grid. Since the grid uses `auto-fill` with `minmax`, the extra div is the grid item now instead of the `<button>`. This is fine — the button inside still fills its container, and the testid wrapper doesn't add any styling that would break layout.

## Bottom line

Safe to merge — this is a test-infrastructure-only change that adds stable DOM hooks and matching component tests, with no runtime behavior or security impact.

## Quick Summary
*Generated: 2026-05-22T08:23:43.916Z*

**What it does** — Adds a proactive rate limiter for cloud embedding requests (OpenAI, OpenHuman/Voyage, remote custom endpoints). Previously the app would blast requests and absorb 429 errors from the backend's 60/min cap. Now it paces outbound requests at the configured rate using a process-global token bucket keyed by endpoint URL. Local/loopback endpoints (Ollama, LocalAI) are exempt. Configurable via `memory.embedding_rate_limit_per_min` in config or `OPENHUMAN_MEMORY_EMBED_RATE_LIMIT` env var. Default 60/min, `0` disables.

**Safety & Breaking concerns** — **Low risk.**

- **No security issues.** Malformed URLs default to throttled (not exempt), which is the safe direction. No new auth surfaces, no user input handling, no secrets logged.
- **No breaking changes.** Additive config field with `#[serde(default)]` — existing `config.toml` files deserialize fine. No public API or embedding signature changes. The reactive 429 retry/backoff is preserved as a backstop.
- **Edge cases are handled well.** `limit == 0` disables cleanly. Loopback detection covers `localhost`, `127.x.x.x`, `::1`. Bucket capacity is 1 (no burst), which correctly prevents exceeding a hard rolling-minute cap. Poisoned mutex is recovered via `into_inner`. `per_minute.max(1)` guards against divide-by-zero.
- **Minor concern:** The global `BUCKETS` registry uses `OnceLock<Mutex<HashMap>>` — the mutex is held briefly (only for map lookup/insert), and the tokio mutex in `TokenBucket::acquire` is dropped before sleeping, so no deadlock risk. Tests that mutate the global `CONFIGURED_LIMIT` could theoretically race in parallel test runs, but the `set_and_read_round_trip` test restores the default, and `acquire_with_limit` tests use the explicit-limit path to avoid this.
- **One nit:** The `acquire_embedding_slot` call in `openai.rs` is placed after the empty-batch check but before the HTTP call — correct placement.

**Bottom line** — Safe to merge: it's a well-isolated, additive, thoroughly-tested throttle on an internal code path with no API/config breaking changes and sensible defaults.

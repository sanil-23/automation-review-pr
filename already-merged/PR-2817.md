## AI Summary
- **What it does**: Adds `reasoning_content` as a first-class field through the OpenAI-compatible inference stack (`ChatResponse`, `NativeMessage`, `build_native_assistant_history`, `convert_messages_for_native`). For DeepSeek thinking mode and similar reasoning models, the chain-of-thought emitted alongside tool calls is now preserved and replayed verbatim on the follow-up request, fixing the 400 error (Sentry TAURI-RUST-4KB) that caused every multi-turn `deepseek-reasoner` tool call to fail. Non-reasoning models are unaffected — the field is `None` and gated with `skip_serializing_if`.
- **Breaking risk**: Low. The wire format change is additive and guarded — `skip_serializing_if = "Option::is_none"` ensures non-reasoning providers never see the field. The struct-literal additions in tests are mechanical Rust compliance, not logic changes.
- **Security risk**: Zero. No auth changes, no user input surfaces, no secrets handling, no network topology changes.
- **Bottom line**: Safe to merge — pending CI green and streaming path confirmation.

**What it does**

This PR adds a CI safety net to make sure the zh-CN (Simplified Chinese) locale actually ships in production builds. Previously, translations existed in source but were silently stripped from release bundles (issue #2117). The PR:

1. Adds a new script (`verify-i18n-bundle.mjs`) that scans the built JS assets for the `zh-CN` key and the `简体中文` label — fails the build if either is missing.
2. Wires it up as `pnpm i18n:bundle:check` for release validation.
3. Fixes the Chinese label from the generic `中文` to the more accurate `简体中文` ("Simplified Chinese") in the language picker.

**Safety & Breaking concerns** — **Low**

- **No security issues.** The script only reads local files, uses no network, no user input beyond a `--dist` flag with proper validation. No injection surface.
- **No breaking changes to runtime behavior.** The label change from `中文` to `简体中文` is a minor UI text correction — standard practice for distinguishing Simplified from Traditional Chinese. The script is CI-only and doesn't touch app code.
- **Edge cases are handled well.** Missing dist dir, no JS files, unknown CLI args — all produce clear errors with non-zero exits. The `listJsFiles` recursive walker is straightforward.
- **One minor nit:** The recursive file walker doesn't handle symlink loops, but `app/dist` is a build output dir where symlinks are effectively impossible — not a real concern.
- **The unicode escapes match.** `\u7b80\u4f53\u4e2d\u6587` correctly encodes `简体中文`, and the script checks both the literal and escaped forms to handle different bundler output styles. Good.

**Bottom line**

Safe to merge — it's a read-only CI guard script plus a one-word label fix, with no runtime behavior changes.

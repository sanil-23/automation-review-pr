**What it does** — Replaces the `react-ga4` npm package with raw `gtag.js` script injection for Google Analytics, because the library silently fails inside Tauri's CEF webview (desktop app). Adds OpenPanel as a second analytics provider so every page view and event fires to both GA4 and OpenPanel simultaneously. Also disables Sentry performance tracing (was sampling 10% of transactions) — now it only captures errors. Removes the dev-mode guard so analytics can be tested locally.

**Safety & Breaking concerns** — **Low risk**

- **No security flaws found.** Consent gating and the event allowlist are preserved and apply equally to both providers. No PII leakage paths introduced. Ad personalization signals still disabled for GA.
- **Hardcoded OpenPanel client ID** (`e9c996d5-...`) and API URL (`panel.tinyhumans.ai`) in source — not a secret (client-side analytics IDs are public by design), but worth noting it's not configurable via env var like GA is. Minor inconsistency, not a risk.
- **`GA_FORCE_DEV` config added but never read** in `analytics.ts` — the old `IS_DEV` guard was removed entirely, so `GA_FORCE_DEV` is dead code. No harm, but it's unused.
- **OpenPanel `op1.js` loaded from `openpanel.dev`** — this is a third-party CDN dependency. If that domain goes down or is compromised, it could inject arbitrary JS. The PR body says "no new external network dependencies" — that's inaccurate; `op1.js` is new. Low practical risk but worth a CSP review.
- **No breaking changes** — `initGA()`, `trackPageView()`, `trackEvent()` keep the same signatures. `GA_ALLOWED_EVENTS` renamed to `ALLOWED_EVENTS` (breaking if imported elsewhere — but grep shows no external consumers).
- **Sentry tracing disabled permanently** (`tracesSampleRate: 0`) — intentional, but means zero performance visibility going forward. Not a code risk, but an operational trade-off.
- **Tests updated properly** — 24 tests cover init, consent, allowlist, idempotency. The old `react-ga4` mock is cleanly replaced with script-injection assertions.

**Bottom line** — Safe to merge: additive change with preserved privacy guarantees and good test coverage; the only nit is the unused `GA_FORCE_DEV` config and the undeclared new CDN dependency on `openpanel.dev`.

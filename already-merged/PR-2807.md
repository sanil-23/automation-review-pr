## AI Summary
- **What it does**: Wraps the `config.toml` read inside `load_config` with the existing `retry_with_backoff_async` helper to handle Windows transient file-locking errors (ERROR_SHARING_VIOLATION/32, ERROR_ACCESS_DENIED/5, ERROR_DELETE_PENDING/303) that fire when `Config::save`'s atomic rename races a concurrent read. Also upgrades the error message to include the config file path for Sentry triageability. ~8077 Sentry events eliminated.
- **Breaking risk**: Zero — same function signatures, same behavior on macOS/Linux, retry only engages on Windows-specific locking codes.
- **Security risk**: Zero — retry logic on local file reads, no new attack surface.
- **Bottom line**: Safe to merge.

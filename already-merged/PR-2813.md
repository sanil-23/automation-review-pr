## AI Summary
- **What it does**: Adds five Ollama-specific tool-unsupported error phrases to `is_provider_config_rejection_message` in `config_rejection.rs` so the expected HTTP 400s from capability-discovery retries don't fire Sentry events. The compatible provider in `compatible.rs` already retries the request without tools after the 400; this change stops the first-attempt rejection from being reported as a product bug in Sentry (TAURI-RUST-4K7, 14+ events/turn).
- **Breaking risk**: Zero — purely additive to an existing phrase slice, no API surface changes, no logic changes in the retry path.
- **Security risk**: Zero — no auth, secrets, injection surfaces, or network-facing changes.
- **Bottom line**: Safe to merge once CI is green.
